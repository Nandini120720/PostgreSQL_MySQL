--1. sample db bakup

Example :
mysqldump -u root -p sample_db > sample_db_clean.sql


--2. single table bkup 'users_partitioned' in db 'sample_db'
# Dump only the 'users_partitioned' table
mysqldump -u root -p sample_db users_partitioned > users_partitioned_backup.sql

# Create 'new_db' if it doesn't exist (on root)
mysql -u root -p -e "CREATE DATABASE IF NOT EXISTS new_db;"

# Restore into 'new_db' (imported 'sample_db' bkp in 'new_db'db)
mysql -u root -p new_db < users_partitioned_backup.sql

View a large SQL dump file:
less sample_db_backup.sql     ----use 'less' to check bakup has been done or not 


--3. Backup full database (data only): This backs up only the data (no CREATE TABLE statements).
mysqldump -u root -p --no-create-info sample_db > sample_db_data_only.sql

--4. Backup single table (data only): This will include only INSERT statements, like:
mysqldump -u root -p sample_db users_partitioned --no-create-info > users_partitioned_data_only.sql

--5. Structure Only Backup
To back up only the table structures (no data): ðŸ‘‰ Useful for duplicating database schema (tables, indexes, etc.) without data.
mysqldump -u root -p --no-data sample_db > sample_db_structure_only.sql

--6. Full Backup (Structure + Data)
Default behavior of mysqldump:
mysqldump -u root -p sample_db > sample_db_full.sql



----important----- Backup Steps (mysqldump)

mysqldump -u <username> -p <database_name> > backup.sql
mysqldump -u root -p mydatabase > mydatabase_backup.sql
mysqldump -u root -p --databases db1 db2 db3 > multi_db_backup.sql
mysqldump -u root -p --all-databases > all_db_backup.sql
mysqldump -u root -p mydatabase table1 table2 > table_backup.sql

Backup with Stored Procedures, Triggers, Views, Events    -----------use this always
mysqldump -u root -p --routines --events --triggers mydatabase > full_backup.sql

Backup with Data Consistency (InnoDB)
mysqldump -u root -p --single-transaction --quick --lock-tables=false mydatabase > consistent_backup.sql




# Restore the Backup

To restore any of these backups into a different database (e.g., new_db):

mysql -u root -p new_db < sample_db_data_only.sql     	# For data only
mysql -u root -p new_db < sample_db_structure_only.sql  # For structure only
mysql -u root -p new_db < sample_db_full.sql            # For full backup
